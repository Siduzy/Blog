Setup up git

Install

  • Ubuntu apt-get install git

Set up

  • Config username (best to user your GitHub username) git config --global user.name "Your Name Here"
  • Config email (best to use your GitHub email) git config --global user.email "your_email@example.com"
  • Save password for HTTPS URL (not SSH URL) git config --global credential.helper 'cache --timeout=3600'

Create Repo

Once clicked 'Create a repo' on github, and filled the basic info. You need to do these steps:

  • Create your local repo
  • Add a README.md file
  • Add comments and commit
  • Update the GitHub side

Create local repo

  • Create a folder for Git: mkdir ~/Git
    cd ~/Git
  • Create a folder for your repo: mkdir Man-Bash
    cd Man-Bash
  • Initialize the .git folder at ~/Git/Man-bash: git init
    ls -a

Add a README.md file

  • Create README.md for your repo touch README.md
  • Edit README
  • Add the README to your local repo git add README.md for one file or git add * for all files at this level

Add comments and commit

  • Commit won't upload the file to the remote repo. It remembers the timepoint, only save the changed files, compress the files and so on. git commit -m 'add README'

Update the GitHub side

  • Create a named repo git remote add origin https://github.com/ZhangNaixiao/Man-Bash.git
  • Push the commits to the 'master' branch. git push origin master

On the last setup, your real GitHub account (username/password) will be asked to enter.

Then refresh your repo page on GitHub. You will see the ReadMe.md file is uploaded.

Pull your repo

You can download the remote repo to your local by do this:

  • Tell git which repo you want to download git clone https://github.com/ZhangNaixiao/Man-Bash.git
  • Update it git pull

Branch

A repo can be contributed by many people. So only one branch is not enough. We need more. Follow these steps:

  • See current branches git branch
  • Create a new branch git branch branchTest
  • Change branch location to the new one git checkout branchTest
    git branch
  • Push git push origin branchTest

Now, a new branch will be created on your repo page on GitHub.

  • Clone the new branch(branchTest) to ~/Git/branchTest git clone https://github.com/ZhangNaixiao/Man-Bash.git -b branchTest ~/Git/branchTest
  • Add/change some files to new branch
  • Add these files to local repo git add *
  • Commit git commit -m 'update README.md'
  • Push git push origin branchTest

Rollback

If you commit by mistake, you can revert to a timepoint before, here are two basic tips:

Revert and keep the commit log

  • Revert to a commit code. git revert 7b863499dd7115298ad00bb7d8475d619f223dd9
  • Push git push origin branchTest

The README.md at branchTest will be deleted and there will be a new commit at branchTest branch on your GitHub page.

Revert but not keep the commit log

  • Revert to a commit code. git reset 5b1798ff27a19b0fdeb95230719f5689e3f2df3e --hard
  • Force Push git push origin branchTest -f

Diff and Merge

After do the changes on branchTest, useful changes should be merge to the 'master' branch.

  • Switch to branch 'branchTest' git checkout branchTest
  • See the log of changed file git log -- README.md
  • See the diff git diff <commit1> <commit2> README.md
  • Switch to the branch you want to merge into(here is 'master') git checkout master
  • Merge branch git merge branchTest
  • Push git push origin master

Del the branch

  • See the latest changes git branch -v
  • After ensured nothing to worried, switch to another branch git checkout master
  • Delete the branch git branch -d branchTest
  • Delete the remote branch git push origin :heads/branchTest
